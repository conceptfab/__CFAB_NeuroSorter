Dokumentacja CFAB NeuroSorter

Wprowadzenie

CFAB NeuroSorter to aplikacja, która posiada rozbudowaną mechanikę szkolenia i doszkalania modeli (fine-tuningu). Pozwala zautomatyzować procesy, wykorzystując wsadowe przetwarzanie zadań. Umożliwia wydajne testowanie wielu wariantów ustawień treningu i fine-tuningu modeli. System kategoryzowania obrazów pozwala w efektywny sposób testować modele.

Główne funkcje

  • Klasyfikacja pojedynczych obrazów
  • Przetwarzanie wsadowe dużych zbiorów
  • Trenowanie własnych modeli
  • Zarządzanie modelami
  • Generowanie raportów

Szybki start

  1. Wybierz lub wytrenuj model w zakładce 'Modele'
  2. Użyj zakładki 'Klasyfikacja' do analizy pojedynczych obrazów
  3. Wykorzystaj 'Przetwarzanie wsadowe' do pracy z większymi zbiorami
  4. Generuj raporty w zakładce 'Raporty'

Parametry treningu modeli sieci neuronowych

Trening od podstaw vs Fine-tuning

Trening od podstaw

Trening od podstaw (from scratch) polega na trenowaniu modelu z losowo zainicjalizowanymi wagami. Jest to proces wymagający dużych zasobów obliczeniowych i dużych zbiorów danych treningowych.

Fine-tuning

Fine-tuning (doszkalanie) polega na dostosowaniu już wytrenowanego modelu do nowego zadania poprzez aktualizację wag modelu na nowych danych. Jest to mniej wymagające obliczeniowo i pozwala osiągnąć dobre wyniki nawet z mniejszą ilością danych.

Parametry treningu

Architektura modelu (model_arch)

Wartość Trening od podstaw Fine-tuning Opis
resnet50 ✓ ✓ Model o dobrej równowadze między wydajnością a dokładnością. Dobrze sprawuje się w wielu zadaniach klasyfikacji.
efficientnet ✓ ✓ Efektywna architektura zaprojektowana pod kątem równowagi wydajności i zasobów. Zalecana dla większości zadań.
mobilenet ✓ ✓ Lekka architektura zoptymalizowana pod kątem urządzeń mobilnych. Dobra gdy priorytetem jest szybkość.
vit ✓ ✓ Vision Transformer - nowoczesna architektura wykorzystująca mechanizm uwagi. Dobra dla złożonych zadań, wymaga więcej danych.
convnext ✓ ✓ Nowoczesna architektura konwolucyjna o wysokiej wydajności. Dobra dla złożonych zadań klasyfikacji.

Liczba epok (epochs)

Kontekst Zalecana wartość Opis
Trening od podstaw 30-100 Wyższa liczba epok pozwala na lepsze wytrenowanie modelu od podstaw. Zalecane 50-100 dla złożonych zadań.
Fine-tuning 10-30 Mniejsza liczba epok jest wystarczająca przy doszkalaniu istniejącego modelu. Zalecane 10-20 dla większości przypadków.

Rozmiar wsadu (batch_size)

Kontekst Zalecana wartość Opis
Trening od podstaw Zależna od GPU Większe wartości przyspieszają trening, ale wymagają więcej pamięci VRAM.
Fine-tuning Zależna od GPU Podobnie jak w treningu od podstaw, ale można użyć większych wartości dzięki mniejszej złożoności obliczeniowej.

Typowe wartości dla różnych GPU:

  • 16-32: GPU z 4GB VRAM
  • 32-64: GPU z 8GB VRAM
  • 64-128: GPU z 12GB VRAM
  • 128-256: GPU z 16GB+ VRAM

Współczynnik uczenia (learning_rate)

Kontekst Zalecana wartość Opis
Trening od podstaw 1e-3 do 1e-4 Wyższe wartości pozwalają na szybsze uczenie się modelu, ale mogą prowadzić do niestabilności.
Fine-tuning 1e-5 do 1e-6 Niższe wartości zapewniają subtelne dostrajanie wag modelu, zachowując wcześniej nauczone cechy.

Optymalizator (optimizer)

Wartość Trening od podstaw Fine-tuning Opis
Adam ✓ ✓ Uniwersalny optymalizator z adaptacyjnymi współczynnikami uczenia. Dobry wybór dla większości przypadków.
SGD ✓ ✓ Stochastyczny spadek gradientu. Może dawać lepsze wyniki przy odpowiednim dostrojeniu, ale trudniejszy w użyciu.
AdamW ✓ ✓ Wersja Adam z lepszą regularyzacją wag. Zalecany dla modeli narażonych na przeuczenie.
RMSprop ✓ ✓ Dobry w zadaniach z dużą zmiennością gradientów.

Harmonogram uczenia (scheduler)

Wartość Trening od podstaw Fine-tuning Opis
None ✓ ✓ Brak harmonogramu - stały współczynnik uczenia.
StepLR ✓ ✓ Zmniejsza learning rate o stały współczynnik co określoną liczbę epok.
CosineAnnealingLR ✓ ✓ Płynnie zmienia learning rate według funkcji kosinusowej.
ReduceLROnPlateau ✓ ✓ Zmniejsza learning rate gdy określona metrika przestaje się poprawiać. Zalecane dla większości przypadków.
OneCycleLR ✓ ✗ Zaawansowany harmonogram ze zmiennym learning rate w trakcie treningu. Dobry dla treningu od podstaw.

Regularyzacja wag (weight_decay)

Kontekst Zalecana wartość Opis
Trening od podstaw 1e-4 do 1e-5 Zapobiega przeuczeniu modelu poprzez karanie zbyt dużych wag.
Fine-tuning 1e-5 do 1e-6 Mniejsze wartości zapobiegają nadmiernym zmianom w już wyuczonych wagach.

Przycinanie gradientu (gradient_clip_val)

Kontekst Zalecana wartość Opis
Trening od podstaw 0.5 - 1.0 Zapobiega eksplozji gradientów, stabilizując trening.
Fine-tuning 0.1 - 0.5 Mniejsze wartości zapewniają bardziej subtelne aktualizacje.

Cierpliwość wczesnego zatrzymania (early_stopping_patience)

Kontekst Zalecana wartość Opis
Trening od podstaw 8-15 Dłuższa cierpliwość pozwala modelowi przejść przez potencjalne plateau podczas treningu.
Fine-tuning 5-10 Krótszy okres cierpliwości, ponieważ fine-tuning powinien szybciej osiągnąć optimum.

Podział walidacyjny (validation_split)

Kontekst Zalecana wartość Opis
Trening od podstaw 0.1 - 0.2 Określa, jaka część danych treningowych będzie używana do walidacji podczas treningu.
Fine-tuning 0.1 - 0.2 Podobnie jak przy treningu od podstaw.

Augmentacja danych (augmentation)

Wartość Trening od podstaw Fine-tuning Opis
basic ✓ ✓ Podstawowe przekształcenia (obrót, przesunięcie, odbicie). Zalecane dla większości przypadków.
advanced ✓ ○ Zaawansowane przekształcenia (perspektywa, zniekształcenia). Przydatne przy małych zbiorach danych.

Mieszana precyzja (mixed_precision)

Kontekst Zalecana wartość Opis
Trening od podstaw True dla kart NVIDIA RTX Przyspiesza trening poprzez używanie obliczeń w niższej precyzji, zachowując dokładność.
Fine-tuning True dla kart NVIDIA RTX Podobnie jak w treningu od podstaw.

Zamrożenie warstw (freeze_backbone) - tylko dla fine-tuningu

Kontekst Zalecana wartość Opis
Fine-tuning True Zamrożenie głównej części modelu, trenowanie tylko ostatnich warstw. Zalecane dla małych zbiorów danych.
Fine-tuning False Trenowanie całego modelu. Lepsze dla większych zbiorów danych lub znacząco różniących się od oryginalnych.

Zalecane konfiguracje

Dla treningu od podstaw:

{
  "model_arch": "efficientnet",
  "epochs": 50,
  "batch_size": 64,
  "learning_rate": 1e-3,
  "optimizer": "AdamW",
  "scheduler": "ReduceLROnPlateau",
  "weight_decay": 1e-4,
  "gradient_clip_val": 0.5,
  "early_stopping_patience": 10,
  "validation_split": 0.2,
  "augmentation": {
    "basic": true,
    "advanced": true
  },
  "use_mixed_precision": true
}

Dla fine-tuningu:

{
  "model_arch": "efficientnet",
  "epochs": 15,
  "batch_size": 64,
  "learning_rate": 1e-5,
  "optimizer": "AdamW",
  "scheduler": "ReduceLROnPlateau",
  "weight_decay": 1e-5,
  "gradient_clip_val": 0.1,
  "early_stopping_patience": 5,
  "validation_split": 0.2,
  "augmentation": {
    "basic": true,
    "advanced": false
  },
  "use_mixed_precision": true,
  "freeze_backbone": true
}

Uwagi dodatkowe

  • Wartości parametrów należy dostosować do konkretnego zadania i dostępnych zasobów sprzętowych.
  • Monitoruj metryki walidacyjne podczas treningu, aby wykryć potencjalne problemy (przeuczenie, niedouczenie).
  • Dla najlepszych wyników, eksperymentuj z różnymi konfiguracjami parametrów.
  • Liczba klas jest automatycznie wykrywana na podstawie struktury katalogów z danymi treningowymi.

Uwaga: Kategorie treningu są pobierane automatycznie z nazw folderów w katalogu treningowym. Każdy folder pierwszego poziomu reprezentuje jedną kategorię.
